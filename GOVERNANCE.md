# Governance / 治理模式

## Decision Making

This project uses a **Benevolent Dictator** governance model during its early stage:

- **Project Lead**: [@laiyanlong](https://github.com/laiyanlong)
- **Organization**: [DappGo](https://dappgo.com) — AI Automation for Enterprise

### How Decisions Are Made

1. **Minor changes** (typos, docs, small fixes): Merged by any maintainer after review
2. **Features & enhancements**: Discussed in Issues or Discussions first, then PR reviewed by project lead
3. **Architecture changes**: Require discussion + approval from project lead
4. **Breaking changes**: Announced in Discussions with minimum 7-day comment period

### Becoming a Maintainer

Active contributors who demonstrate:
- 5+ merged PRs
- Consistent code quality
- Helpful community participation

...will be invited to become maintainers with merge permissions.

### Conflict Resolution

1. Discussion in the relevant Issue or PR
2. If unresolved, project lead makes the final call
3. All decisions will include a written rationale

## 決策方式

本專案在早期階段採用 **仁慈獨裁者** 治理模式：

- **專案負責人**：[@laiyanlong](https://github.com/laiyanlong)
- **組織**：[DappGo](https://dappgo.com) — 企業級 AI 自動化

### 決策流程

1. **小型修改**（錯字、文件、小修正）：審查後由任何維護者合併
2. **功能與改進**：先在 Issues 或 Discussions 討論，再由專案負責人審查 PR
3. **架構變更**：需要討論 + 專案負責人批准
4. **破壞性變更**：在 Discussions 公告，最少 7 天評論期

### 成為維護者

活躍貢獻者若展現以下特質將受邀成為維護者：
- 5+ 個已合併的 PR
- 穩定的程式碼品質
- 積極的社群參與
