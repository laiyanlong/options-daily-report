# Report a data quality issue

If a report has wrong data or a field is missing, don't edit the file —
open an issue.

## Why not edit

Reports are regenerated by the private pipeline. Any manual edit gets
overwritten on the next run. The only fix is upstream, in the engine.

## Template

```
Title: [data] <symbol> <what's wrong>
Labels: data-quality

### Report date
YYYY-MM-DD

### Affected symbol(s)
TSLA, AMZN, …

### What's wrong
Precise quote of the incorrect line or field.

### What it should be (if you know)
Source: Yahoo Finance / broker / other

### Impact
How does this mislead a reader?
```

Maintainer will route to the private engine repo for a fix.
